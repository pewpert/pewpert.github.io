01 Productivity · Visualization

Task Flow
Visualizer

Mapping project dependencies before they become bottlenecks.

The What

Task Flow Visualizer is a browser-based tool that lets you define tasks, link them with dependency arrows, and see in a single view where your critical path actually runs. You input tasks as nodes, set "depends on" relationships, and the tool renders an auto-laid-out directed graph — no whiteboard, no sticky notes.

The goal was minimal friction: paste a simple structured list and get a visual map back instantly. No sign-up, no cloud sync, no settings panel bloat. Just the diagram.

The Why

I kept running into the same situation across projects: dependencies existed in everyone's heads, never on paper. By the time a blocker surfaced in a status call, the delay had already rippled two sprints forward.

I tried Jira, Asana, and a few Miro templates — but they all required too much setup for the clarity I needed at the start of a sprint. I wanted something I could pull up in a planning meeting and fill in live. So I built it.

What I Learned

  • Graph layout is a hard problem. I assumed I could write a layout algorithm in an afternoon. Three days later I was reading academic papers on Sugiyama-style layered graphs. I ended up using a library — but understanding why made me a better collaborator with the engineers I work with.

  • Users want to paste, not type. Early testers ignored my form inputs entirely and tried to paste spreadsheet data straight into the textarea. That told me everything about where the friction was.

  • Visual fidelity and functional accuracy aren't the same thing. A beautiful diagram that mislabels the critical path is worse than a plain text list. I learned to test the logic independently of the visuals.
